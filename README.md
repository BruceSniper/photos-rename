# 图片批量重命名工具

一个用 Go 语言编写的简单实用工具,用于将目录中的图片文件按顺序重命名为连续数字格式(如 0001.jpg、0002.jpg)。

## 功能特点

- 支持多种图片格式:jpg、jpeg、png、gif、bmp、webp
- 自动按文件名排序后重命名
- 支持自定义起始编号,方便多个文件夹接续处理
- 支持自定义文件名数字位数,应对大量文件场景
- 提供预览模式,安全查看重命名计划
- 使用临时文件名避免重命名冲突
- 自动跳过已经符合格式的文件

## 使用方法

```bash
go run main.go <目录路径> [起始编号] [digits=位数] [preview]
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| 目录路径 | 是 | 图片所在目录 |
| 起始编号 | 否 | 文件名起始编号,默认为 1 |
| digits=N | 否 | 文件名数字位数,默认为 4 |
| preview | 否 | 预览模式,不实际执行重命名 |

参数顺序不限,可以随意排列。

### 示例

```bash
# 默认从 0001 开始,4 位数字
go run main.go ./photos

# 从 2001 开始
go run main.go ./photos 2001

# 使用 6 位数字: 000001, 000002, ...
go run main.go ./photos digits=6

# 6 位数字 + 起始编号 2001: 002001, 002002, ...
go run main.go ./photos 2001 digits=6

# 预览模式
go run main.go ./photos 2001 digits=6 preview
```

### 编译后使用

```bash
# 编译
go build -o photo-rename main.go

# 预览
./photo-rename ./photos preview

# 执行重命名
./photo-rename ./photos 2001
```

## 使用场景

假设你有两个文件夹的图片需要处理,希望编号连续:

```bash
# 第一个文件夹: 0001 ~ 2000
go run main.go ./folder1

# 第二个文件夹: 从 2001 继续
go run main.go ./folder2 2001
```

如果文件数量超过 9999,可以增加位数:

```bash
# 使用 6 位数字,支持最多 999999 个文件
go run main.go ./photos digits=6
```

## 使用示例

假设你的 `photos` 目录中有以下文件:
```
004.jpg
007.jpg
012.jpg
```

### 1. 先预览

```bash
go run main.go ./photos preview
```

输出:
```

----------------------------------------

这是预览模式,没有实际执行重命名操作
```

```bash
go run main.go ./photos
```

输出:
```

找到 3 个图片文件

重命名计划:
----------------------------------------

007.jpg -> 0002.jpg

```

## 注意事项

1. **建议先使用预览模式**查看重命名计划
2. **重要文件请先备份**,虽然程序经过测试,但操作文件系统总存在风险
3. 程序会按字母顺序排序后重命名,请确认排序结果符合预期
4. 保留原文件扩展名

## 工作原理

1. 扫描指定目录,获取所有图片文件
2. 按文件名排序
3. 根据起始编号和位数生成重命名计划
4. 先将所有文件重命名为临时文件名(避免冲突)
5. 再将临时文件名改为最终文件名
